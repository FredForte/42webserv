webserv test panel

Mirrors tests/e2e.sh. Same-origin tests on this port () run live and show the real status + body. Tests needing a custom Host header or a different port can't run from a browser — they show why, plus a curl to run by hand.

What your browser actually sends

A page can't read the headers the browser puts on the wire (Host, User-Agent, Accept, …) — fetch() only exposes headers you set yourself. So this sends a request to /cgi-bin/reflect.py, which rebuilds it from what webserv parsed and echoes it back: the real request line, headers, and body as the server saw them.
